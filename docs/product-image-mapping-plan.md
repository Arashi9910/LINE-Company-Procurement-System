# 執行計畫：飛鼠商品圖片對照資料

## 架構決策

- 使用獨立 `商品圖片對照` 分頁，避免在資料尚未驗證前改動正式 `SKU主檔`。
- 保留飛鼠 912 筆原始列，透過狀態欄呈現重複、缺圖與未綁定狀況。
- 商品首圖使用銷售商品編號組成的穩定網址；列表上的雜湊圖片視為規格圖片。

## 任務

### Task 1：擷取與正規化飛鼠列表

**Acceptance**
- 以 `pageSize=1000` 單頁取得 912 筆，避免非唯一排序造成分頁邊界重複。
- 每筆包含貨品 ID、SKU、商品名稱、規格、銷售商品資訊與圖片網址。
- 擷取後回到列表第 1 頁。

**Verify**
- 驗證 912 個不重複飛鼠貨品 ID。
- 統計來源圖片類型及例外筆數。

**Dependencies**：無。

### Task 2：讀取 SKU 主檔並建立配對結果

**Acceptance**
- 以 `SKU主檔!A2:A913` 的實際資料進行精確配對。
- 重複 SKU 與未配對 SKU 保留並標示。

**Verify**
- 配對筆數、重複筆數與未配對筆數加總可解釋 912 筆來源列。

**Dependencies**：Task 1。

### Task 3：建立並寫入商品圖片對照分頁

**Acceptance**
- 新增 1 個 1000 × 15 的 `商品圖片對照` 分頁。
- 寫入 15 欄表頭與 912 筆資料。
- 凍結表頭、加入篩選器並套用既有試算表的簡潔樣式。

**Verify**
- 回讀 `A1:O5`、`A910:O913`。
- 中繼資料顯示分頁存在且網格足夠。

**Dependencies**：Tasks 1、2。

### Checkpoint：資料底稿完成

- 飛鼠後台資料未修改。
- `SKU主檔` 未修改。
- 912 筆圖片對照可供下一階段 LIFF 卡片介面使用。

## 風險與處理

| 風險 | 處理 |
| --- | --- |
| 相同更新時間造成分頁邊界重複或漏列 | 使用 `pageSize=1000` 單頁擷取，驗證 912 個不重複貨品 ID |
| 圖片網址存在但日後失效 | 保留來源網址與狀態；前端實作替補圖 |
| SKU 重複造成錯配 | 不去重，明確標示 `重複SKU` |
| 未綁定銷售商品無法取得主圖 | 使用規格圖，無規格圖則標示 `待補圖片` |
