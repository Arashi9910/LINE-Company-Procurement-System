# LINE 群組指令功能規格

## 目標

讓公司 LINE 工作群組可以直接查詢補貨案件狀態，並由既有管理員透過 `@成員` 管理系統權限，不必每次開啟 Google 試算表。

成功條件：

- 工作群內可用六種狀態查詢指令，每次最多顯示 10 筆案件並附總筆數。
- 每筆以「補貨申請單」彙整，不逐一列出全部 SKU。
- 管理員可用群組指令授權、停用及查詢成員權限。
- 授權異動會寫入既有「操作紀錄」。
- 其他 LINE 群組不能使用指令，也不能覆寫目前公司工作群組設定。
- 不破壞既有「補貨」LIFF 表單、下單確認、到貨登記及提醒功能。

## 技術堆疊

- Node.js 22、ES Modules
- Express 5
- `@line/bot-sdk` 10.2
- Google Sheets API v4
- Node.js 內建測試工具 `node:test`
- Google Cloud Run（本功能完成後另行確認才部署）

不新增 npm 套件，也不新增試算表工作表或欄位。

## 使用者指令

### 查詢與說明

| 指令 | 結果 |
| --- | --- |
| `查未結案` | 待確認、已下單及部分到貨案件 |
| `查待確認` | 尚未確認下單的案件 |
| `查已下單` | 已確認下單、尚未有到貨紀錄的案件 |
| `查部分到貨` | 已有部分品項或數量到貨的案件 |
| `查已完成` | 最近完成的案件 |
| `查取消` | 最近取消的案件 |
| `補貨指令` | 顯示可用指令及使用方式 |

查詢結果依申請時間新到舊排序，最多顯示 10 筆，第一行顯示符合條件的總筆數。每筆包含申請單號、申請人、品項數與第一個品項簡稱。

### 授權管理

| 指令 | 結果 |
| --- | --- |
| `授權 @小明 採購確認` | 建立或更新為採購確認角色並啟用 |
| `授權 @小明 到貨確認` | 建立或更新為到貨確認角色並啟用 |
| `授權 @小明 管理員` | 建立或更新為管理員並啟用 |
| `授權 @小明 申請人` | 建立或更新為一般申請人並啟用 |
| `停用 @小明` | 停用成員權限，保留原角色 |
| `查權限 @小明` | 顯示成員目前角色與啟用狀態 |

授權類指令只接受 LINE 訊息中的實際 `@成員` mention，不從畫面文字猜測對象。LINE Webhook 必須提供該 mention 的 `userId`；若成員尚未同意 LINE 提供識別資訊，系統會回覆可執行的處理提示。

## 狀態彙整規則

同一申請單可能有多個 SKU，畫面依下列優先順序判定整張申請單狀態：

1. 全部明細皆為「取消」：取消。
2. 全部明細皆為「已完成」或「取消」，且至少一項完成：已完成。
3. 任一明細為「部分到貨」，或同單同時存在已完成與尚未完成明細：部分到貨。
4. 任一明細為「待確認」：待確認。
5. 其餘有效明細皆已下單：已下單。

「未結案」包含待確認、已下單與部分到貨，不包含已完成及取消。

## 權限與安全邊界

- 狀態查詢與 `補貨指令` 可由公司工作群組內任何成員使用。
- 授權、停用及查權限僅限「已啟用的管理員」。
- 管理員不能停用自己，也不能把自己降級，以避免系統失去管理員。
- 授權寫入既有「授權人員」工作表；異動同時寫入「操作紀錄」，包含操作者、對象、角色、動作與時間。
- 以 LINE `webhookEventId` 建立冪等鍵；LINE 重送同一事件時不能重複寫入。
- 若「通知群組 ID」已有值，其他群組只能收到未授權提示，不能更新該設定。
- Webhook 繼續以原始 request body 驗證 `x-line-signature`。

## 專案結構

```text
src/
  routes/webhook.js              # 驗證事件、限制群組、協調回覆
  line/messenger.js              # LINE 回覆與群組成員資料查詢
  services/group-commands.js     # 指令解析、狀態彙整、權限命令
  sheets/repository.js           # 試算表讀取、授權異動與操作紀錄
test/
  group-commands.test.js         # 純邏輯與命令服務測試
  webhook.test.js                # Webhook 整合與工作群限制測試
  sheets-repository.test.js      # Sheets 範圍、寫入與冪等測試
```

## 程式風格

沿用現有 ES Modules、具名匯出、async/await 與依賴注入，不引入全域單例。Webhook 保持薄層，狀態判斷寫成可單獨測試的純函式。

```js
export function parseGroupCommand(message) {
  const text = message?.type === 'text' ? message.text.trim() : '';
  if (text === '查未結案') return { type: 'status', status: '未結案' };
  return null;
}
```

## 測試策略

- 先寫失敗測試，再完成最小實作。
- 單元測試覆蓋六種狀態、混合 SKU 狀態、10 筆限制與權限指令解析。
- Repository 測試覆蓋授權新增、更新、停用、操作紀錄、冪等重送與不覆寫群組。
- Webhook 測試覆蓋公司群、其他群、私人對話、非管理員、無有效 mention 與既有「補貨」指令。
- 完成後執行 `npm test`、`npm run lint`、`npm run build`。

## 非本次範圍

- 不允許用文字輸入或 LINE 顯示名稱搜尋成員，避免授權錯人。
- 不提供跨群組管理或第二個工作群組。
- 不改角色可執行的既有下單／到貨權限規則。
- 不在這次開發中自動部署正式環境。

## 已知限制

- LINE 只會在成員同意提供識別資訊時，於 mention 中提供 `userId`；否則管理員需請該成員先與官方帳號互動並同意後再試。
- 查詢結果為訊息當下從 Google Sheets 讀取，不是即時庫存系統資料。

## 未決問題

無。上述指令與權限規則已由使用者於 2026-07-14 同意。
