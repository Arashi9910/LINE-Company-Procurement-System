# LIFF 商品卡片式補貨表單實作計畫

1. 在 Sheets repository 新增圖片對照讀取，依 SKU 合併到可補貨清單；若圖片分頁缺漏則安全降級。
2. 為圖片欄位解析、未結案數合併及缺漏情境補上後端測試。
3. 建立可單元測試的前端 catalog 模組，負責商品分組、搜尋與數量統計。
4. 重構 LIFF 補貨首頁：商品卡片、規格底部面板、數量調整與固定清單工具列。
5. 保留原本的補貨送出、確認下單與到貨確認資料流。
6. 執行測試、語法檢查、建置檢查，再以瀏覽器驗證響應式版面與互動。
7. 部署至既有 Cloud Run 服務，確認健康狀態與 LIFF 正式網址後合併回主分支。
