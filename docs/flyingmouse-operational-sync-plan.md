# 執行計畫：飛鼠圖片與庫存快照自動同步

## Task 1：建立飛鼠貨品列表擷取器

- 共用既有 Playwright 登入流程。
- 固定 `pageSize=1000`，依連結與元素結構解析貨品 ID、SKU、商品、規格與圖片。
- 驗證 DOM SKU 與官方 Excel 完全一致。
- 單元測試涵蓋規格圖、商品首圖、預設圖、未綁定與重複資料。

## Task 2：建立商品圖片對照差異同步

- 驗證 15 欄表頭與既有穩定鍵。
- 只新增或更新自動管理列；保護人工來源列；不刪資料。
- 只在 A:N 有變更時更新 O 欄時間。
- 容量不足時安全擴增列數。
- 單元測試驗證精確 ranges、冪等與人工保護。

## Task 3：建立 SKU 庫存快照同步

- 即時讀取 `SKU主檔!A:E` 並驗證表頭與唯一 SKU。
- 以官方 Excel 為來源，配對率低於 90% 時停止。
- 僅批次更新有差異的 E 欄。
- 單元測試驗證零差異、部分差異、低覆蓋率與重複 SKU。

## Task 4：整合每日 Cloud Run Job

- browser-export + review 模式才執行圖片同步。
- review 模式在核准新品處理後更新庫存與圖片對照。
- 離線模式維持可測試，不嘗試登入或同步圖片。
- JSON 報告記錄兩項同步摘要。

## Task 5：驗證與部署

- 執行完整測試、lint 與 build。
- 本機唯讀擷取 932 筆並確認與官方 Excel SKU 集合一致。
- 部署前再次確認正式寫入範圍；部署後手動執行一次 Cloud Run Job。
- 驗證 `商品圖片對照` 新增新品、`SKU主檔` 僅 E 欄有庫存差異寫入、排程仍為每日 03:00。
