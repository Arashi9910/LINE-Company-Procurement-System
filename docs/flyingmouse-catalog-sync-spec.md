# 規格：飛鼠新品目錄唯讀同步

## 目標

以獨立批次程式登入飛鼠後台，使用官方「貨品列表 Excel 匯出」，驗證並解析貨品資料，再與既有目錄產生差異預覽。此流程完成後不依賴 Codex 或 Claude 執行，也不消耗模型 Token。

第一階段只讀取飛鼠資料並產生本機報告，不寫入飛鼠、不修改 Google Sheet、不自動新增或刪除商品。

## 固定流程

1. 讀取 `.env.flyingmouse-login.txt`；不得輸出帳號、密碼或瀏覽器儲存狀態。
2. 使用全新無痕瀏覽器內容登入飛鼠後台。
3. 開啟貨品列表並觸發官方 Excel 匯出。
4. 將檔案下載至 `work/flyingmouse/downloads/`。
5. 驗證工作表、表頭、貨品編號、庫存及重複鍵。
6. 與指定的既有目錄 Excel 依貨品編號比對。
7. 在 `work/flyingmouse/previews/` 產生 JSON 差異報告後結束。

## 官方匯出資料契約

工作表第一列必須依序為：

| 欄位 | 規則 |
| --- | --- |
| 貨品編號 | 必填、去除前後空白、整份檔案唯一 |
| 商品名稱 | 文字 |
| 規格 1 | 文字，可空白 |
| 規格 2 | 文字，可空白 |
| 庫存 | 非負整數 |
| GTIN | 以文字處理，避免遺失前導零 |
| 儲位 | 文字，可空白 |

表頭不符、來源為空、貨品編號缺漏或重複、庫存無效時，程式必須直接失敗且不產生可供同步的結果。

## 差異報告

- `newItems`：飛鼠存在、既有目錄沒有的貨品編號。
- `changedItems`：貨品編號相同，但商品名稱或規格不同。
- `unchangedItems`：貨品編號與商品描述皆相同。
- `sourceMissingItems`：既有目錄存在、飛鼠匯出沒有；只供人工檢查，絕不自動刪除。
- `conflicts`：既有目錄貨品編號重複或無法唯一配對；只供人工處理。

庫存、GTIN 與儲位會保留在來源快照，但不作為新品目錄的「商品描述已變更」判斷依據。庫存同步屬於另一個需要人工核准與併發檢查的流程。

## 安全限制

- 不提交任何飛鼠表單，不呼叫匯入庫存更新。
- 不保存 Cookie、localStorage、瀏覽器 profile 或登入狀態。
- 不將 `.env.flyingmouse-login.txt`、下載檔及差異報告加入 Git。
- 登入遇到驗證碼、OTP、權限不足或頁面結構改變時安全停止。
- 不接受靜默降級：下載失敗、檔名不符或 Excel 驗證失敗都必須回傳非零結束碼。

## 驗收標準

- 可只用既有 Excel 執行離線解析與差異預覽。
- 可用瀏覽器自動登入並下載官方 `.xlsx`。
- 目前飛鼠匯出可解析為 912 個唯一貨品編號。
- 差異報告可重現新增、重複與來源缺漏統計。
- `npm test`、`npm run lint`、`npm run build` 全部通過。

## 雲端化邊界

本機 PoC 通過後，才將相同命令封裝成獨立 Cloud Run Job，由 Cloud Scheduler 觸發。雲端帳密改存 Secret Manager，下載檔只在單次工作內暫存；第一版雲端工作仍只產生差異預覽，不寫入正式目錄。
