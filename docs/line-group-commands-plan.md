# LINE 群組指令實作計畫

## 1. 指令解析與案件彙整

- 建立 `src/services/group-commands.js`。
- 先測六種查詢、說明、授權語法、有效 mention 與無效 mention。
- 將追蹤列依申請單號彙整，套用狀態優先順序、排序與 10 筆限制。
- 驗收：純函式測試通過，混合 SKU 案件只出現一筆。
- 依賴：無。
- 規模：小。

## 2. Google Sheets 查詢介面

- 在 repository 新增讀取實際追蹤資料列的方法，只讀到最後一筆實際申請單。
- 避免把預填公式空白列當成案件。
- 驗收：測試確認讀取範圍有界且回傳正確欄位。
- 依賴：步驟 1 的資料格式。
- 規模：小。

## 3. 授權異動與操作紀錄

- 在同一個 repository 寫入佇列中完成授權新增／更新／停用。
- 使用 webhook event ID 冪等處理，並以一次 batch update 同步寫入「授權人員」與「操作紀錄」。
- 防止管理員停用或降級自己。
- 驗收：新增、更新、停用、重送及自我鎖定測試通過。
- 依賴：步驟 1 的權限命令格式。
- 規模：中。

## 4. LINE 與 Webhook 整合

- Messenger 新增群組成員資料查詢。
- Webhook 僅處理已知指令，私人對話給群組提示，其他群組拒絕且不覆寫設定。
- 管理員命令呼叫授權服務，所有命令使用原 reply token 回覆一次。
- 驗收：Webhook 簽章、公司群、其他群、角色及 mention 整合測試通過。
- 依賴：步驟 1 至 3。
- 規模：中。

## 5. 完整驗證與交付

- 執行全部測試、lint 與 build。
- 檢查 git diff 不含密鑰、輸出檔或無關重構。
- 彙整實際可用指令、限制與部署變更。
- 正式部署前再次取得使用者確認。
- 依賴：步驟 1 至 4。
- 規模：小。
