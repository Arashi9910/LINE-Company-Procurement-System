# LIFF 商品卡片式補貨表單規格

## 1. 功能需求

- 補貨首頁以「商品」為單位顯示卡片，不再直接平鋪全部 SKU。
- 商品卡片顯示代表圖片、商品名稱、可選規格數與未結案警示。
- 點擊商品卡片後，以行動裝置友善的底部面板顯示該商品全部規格。
- 規格列顯示規格圖片、規格名稱、SKU、庫存快照、未結案數與數量調整器。
- 使用者可在同一商品選擇多個規格，並可跨商品累積本次補貨清單。
- 頁面底部固定顯示已選規格數、總件數與「查看補貨清單」。
- 補貨清單可修改數量、移除品項、填寫整張申請備註並送出。
- 搜尋範圍包含商品名稱、規格、SKU 與既有搜尋關鍵字。
- 既有確認下單、到貨確認、LINE 身分驗證、群組來源驗證及通知流程維持不變。

## 2. 資料與分組規則

- `SKU主檔` 仍是 SKU 是否可補貨、名稱、規格、庫存、單位的主要來源。
- `商品圖片對照` 透過 SKU 與 `SKU主檔` 合併，提供商品編號、商品首圖、規格圖與圖片狀態。
- 優先以 `銷售商品編號` 分組；缺少銷售商品編號時，以正規化後的商品名稱分組；兩者都缺少時，每個 SKU 各自成組。
- 商品卡片代表圖片依序採用商品首圖、組內第一張可用規格圖；都不存在或載入失敗時顯示圖像佔位符。
- 規格圖片依序採用該 SKU 規格圖、商品卡片代表圖片；載入失敗時顯示佔位符。
- 圖片只引用現有 HTTPS 網址，不複製或重新託管飛鼠圖片。
- 圖片對照表不存在或個別 SKU 無資料時，API 仍須回傳可使用的 SKU 清單。

## 3. API 契約

- `GET /api/skus` 保留既有欄位並新增：
  - `productId`
  - `productCode`
  - `variantName`
  - `mainImageUrl`
  - `variantImageUrl`
  - `listImageUrl`
  - `imageStatus`
  - `bindingStatus`
- `POST /api/requests` 輸入格式不變，仍為 `items: [{ sku, quantity }]`。
- 新欄位皆為字串且允許空值，避免舊資料或圖片對照缺漏造成整頁失敗。

## 4. 互動與版面

- 主要設計寬度以 LINE LIFF 手機畫面為準，桌面仍可正常使用。
- 商品卡片使用兩欄網格；窄螢幕維持兩欄但縮小間距與字級。
- 規格選擇面板由畫面底部進入，支援關閉按鈕、點擊遮罩及 Escape 關閉。
- 數量使用「減、數字、加」控制；0 代表未選，增加到 1 即加入，減到 0 即移除。
- 底部固定工具列只在至少選擇一個規格時顯示，並保留 iPhone 安全區間距。
- 補貨清單以同頁區塊呈現，點擊底部工具列後捲動到該區塊。
- 所有動態按鈕具備可讀的 `aria-label`，底部面板使用 dialog 語意。

## 5. 錯誤處理與限制

- API 或 LINE 初始化失敗時沿用頁首狀態訊息顯示原因。
- 圖片載入失敗不顯示破圖，改為佔位符且不影響選購。
- 單次最多 50 個 SKU 的後端限制維持不變；前端在第 51 個規格加入前提示使用者。
- 數量必須為 1 到 999999 的整數。
- 這一版不做分類階層、常用清單、掃碼、圖片上傳及飛鼠即時同步排程。

## 6. 驗收標準

- 有相同銷售商品編號的多個 SKU 只顯示一張商品卡片。
- 搜尋 SKU 或規格時，對應商品卡片仍會出現。
- 從規格面板選擇多個規格後，固定工具列與補貨清單數字正確同步。
- 修改或移除清單品項後，規格面板數量同步更新。
- 送出仍產生一張包含多個 SKU 的補貨單，重複警示與 LINE 推播正常。
- 缺圖、圖片載入失敗及未綁定商品不會阻止表單操作。
- 現有後端測試、語法檢查與建置檢查全部通過。
