# 執行計畫：飛鼠目錄差異審核與新品匯入

## Phase 1：審核資料模型

### Task 1：差異轉換與來源指紋

**Acceptance**

- 新增、描述變更、來源缺漏與衝突可轉成固定欄位。
- 同一差異的同步鍵穩定，來源資料變動時指紋會改變。
- 飛鼠庫存、GTIN、儲位保留在審核資料中。

### Task 2：審核分頁建立與 upsert

**Acceptance**

- 缺少分頁時建立 `飛鼠目錄待確認`、凍結表頭、加入篩選與審核狀態下拉選單。
- 既有分頁表頭必須完全符合資料契約。
- 重跑保留審核備註與有效的人工作業狀態。
- 已不存在的差異只標為 `已解除`，不刪列。

## Checkpoint：只寫審核區

- 單元測試涵蓋建立、重跑、狀態保留、來源變更與解除。
- 零差異正式試跑只建立表頭，不改 `SKU主檔`。

## Phase 2：核准後匯入

### Task 3：重新驗證與冪等檢查

**Acceptance**

- 僅處理 `待處理／新增／核准匯入`。
- 當次來源不存在或指紋不同時改為 `需重新確認`。
- 主檔已有完全相同 SKU 時視為已匯入；已有不同資料時停止該列匯入。

### Task 4：安全寫入 `SKU主檔`

**Acceptance**

- 只寫 A:J、L:M，不碰 K、N 陣列公式。
- 新品預設 `一般SKU`、`是`、`件`、`飛鼠`，核准匯入後立即可補貨。
- 主檔與審核狀態用同一批 values 更新。
- 找不到空白列、表頭不符或有重複 SKU 時不執行主檔寫入。

## Phase 3：雲端部署

### Task 5：明確切換為 review 模式

**Acceptance**

- 預設命令仍為唯讀；只有 `FLYINGMOUSE_SHEET_MODE=review` 才取得 Sheets 寫入 scope。
- 部署腳本與 Cloud Run label 明確顯示 review 模式。
- 既有 LINE Cloud Run Service 不變。

### Task 6：正式零差異驗證

**Acceptance**

- 建立審核分頁後回讀表頭、格式與資料驗證。
- 執行一次 Job，摘要仍為 912 對 912、零差異。
- `SKU主檔` 的資料列數、表頭及 K/N 公式維持不變。
