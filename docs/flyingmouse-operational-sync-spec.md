# 規格：飛鼠圖片與庫存快照自動同步

## 目標

把既有每日 03:00（Asia/Taipei）的飛鼠目錄同步補上兩項作業：

1. 將飛鼠貨品列表的新品圖片與商品關聯資訊同步到 `商品圖片對照`。
2. 將官方 Excel 的最新庫存量同步到 `SKU主檔` 的 `庫存快照` 欄。

LINE 補貨頁重新整理或重新開啟後，直接讀取上述兩個分頁，因此不需要另外複製資料到應用程式資料庫。

## 圖片來源與欄位

- 來源：登入後的 `/admin/part/list/*`，固定使用 `pageSize=1000`。
- 解析依據：
  - `/admin/part/edit/{id}`：飛鼠貨品 ID 與 SKU。
  - `/admin/product/edit/{id}`：銷售商品 ID 與銷售商品編號。
  - 貨品列中的 `<b>`：商品名稱。
  - 商品名稱後的文字：規格。
  - 貨品列中的 `<img>`：列表實際圖片。
- 商品首圖網址仍依銷售商品編號組成：
  `https://img.fslol.com/pic/ss-select/{銷售商品編號}/cover.jpg`。
- `img.fslol.com/10371/...` 視為規格圖；`/img/default-cover.jpg` 視為待補圖片；有規格圖但沒有商品首圖時視為待補主圖。
- 綁定狀態沿用既有值：`已綁定銷售商品` 或 `未綁定銷售商品`。

`商品圖片對照` 以飛鼠貨品 ID 為穩定鍵。同步規則如下：

- 新貨品寫入第一個空白列。
- 既有列只有 A:N 的來源資料改變時才更新，並同時更新 O 欄擷取時間。
- N 欄為 `飛鼠貨品列表` 的列視為自動管理；其他來源值視為人工管理，不覆寫。
- 既有列已綁定銷售商品時，若本輪來源暫時缺少綁定，不清空商品 ID、編號、首圖或綁定狀態；後續來源提供有效綁定時仍可升級或更新。
- 不因飛鼠來源暫時缺列而刪除 Google Sheet 舊資料。
- M 欄依同步當下是否存在於 `SKU主檔` 寫入 `已配對` 或 `未配對`。
- 分頁容量不足時只增加列數，不刪除或搬動既有列。

## 庫存快照

- 真實來源：同一輪下載並驗證完成的飛鼠官方 Excel。
- 目標：`SKU主檔` E 欄 `庫存快照`。
- 配對鍵：精確 SKU。
- 只更新有差異的 E 欄儲存格；不修改名稱、規格、狀態、公式或其他欄位。
- 尚未核准匯入的新品不會先建立主檔列，但會列入未配對來源數量。
- 新品日後核准匯入時，既有匯入流程會先帶入該次庫存；每日同步再持續更新。

## 寫入前安全條件

圖片同步必須全部成立：

- 頁面路徑、標題、資料表表頭與 DOM 結構符合預期。
- DOM 貨品 ID 與 SKU 各自不重複。
- DOM SKU 集合與本輪官方 Excel SKU 集合完全一致。
- `商品圖片對照` 15 欄表頭完全一致，既有飛鼠貨品 ID 不重複。

庫存同步必須全部成立：

- `SKU主檔` A:E 表頭完全一致。
- 主檔與來源 SKU 均不重複。
- 官方 Excel 庫存為非負安全整數。
- 主檔 SKU 對官方 Excel 的配對率至少 90%。

任一條件失敗時丟出錯誤，該同步步驟不寫入。所有動作都必須可重跑且結果一致。

## 輸出與監控

每輪 JSON 報告新增：

- `inventorySync`：主檔數、配對率、更新數、未配對數。
- `imageSync`：擷取數、新增數、更新數、未變更數、人工保護數。

成功標準：

- 無差異重跑時不產生圖片或庫存寫入。
- 新品可新增圖片對照列，但在核准進入 `SKU主檔` 前標示為 `未配對`。
- 既有 SKU 庫存改變後只更新 E 欄，LINE 頁面重新載入即顯示新快照。
